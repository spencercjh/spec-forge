# Phase 4.1: Streaming Support Design

> **Status:** Approved
> **Date:** 2026-03-27
> **Author:** Claude + User

## Overview

Add real-time streaming support for LLM enrichment, allowing users to see AI-generated descriptions as they are generated, similar to `ollama run` or ChatGPT.

## Goals

- Real-time streaming output of LLM responses
- Concurrent batch processing with prefix-based output distinction
- CLI flag to disable streaming when needed
- Leverage langchaingo's `llms.WithStreamingFunc` pattern

## Non-Goals

- RAG-based enrichment context (P4.2)
- Token optimization (P4.3)

## Design

### Provider Interface Refactoring

**File:** `internal/enricher/provider/factory.go`

```go
// Provider defines the interface for LLM providers
type Provider interface {
    // Generate generates a response for the given prompt with optional options
    Generate(ctx context.Context, prompt string, opts ...Option) (string, error)
    // Name returns the provider name
    Name() string
}

// GenerateOptions contains options for generation
type GenerateOptions struct {
    StreamingFunc func(ctx context.Context, chunk []byte) error
}

// Option is a functional option for Generate
type Option func(*GenerateOptions)

// WithStreamingFunc sets a streaming callback function
func WithStreamingFunc(fn func(ctx context.Context, chunk []byte) error) Option {
    return func(o *GenerateOptions) {
        o.StreamingFunc = fn
    }
}

// applyOptions applies options and returns the configured GenerateOptions
func applyOptions(opts ...Option) *GenerateOptions {
    cfg := &GenerateOptions{}
    for _, opt := range opts {
        opt(cfg)
    }
    return cfg
}
```

### Provider Implementation (CustomProvider Example)

**File:** `internal/enricher/provider/custom.go`

```go
// Generate generates a response with optional streaming
func (p *CustomProvider) Generate(ctx context.Context, prompt string, opts ...Option) (string, error) {
    cfg := applyOptions(opts...)

    messages := []llms.MessageContent{
        llms.TextParts(llms.ChatMessageTypeUser, prompt),
    }

    var callOpts []llms.CallOption
    if cfg.StreamingFunc != nil {
        callOpts = append(callOpts, llms.WithStreamingFunc(cfg.StreamingFunc))
    }

    response, err := p.llm.GenerateContent(ctx, messages, callOpts...)
    if err != nil {
        return "", fmt.Errorf("custom provider generation failed: %w", err)
    }

    // Extract text from response
    if len(response.Choices) > 0 {
        return response.Choices[0].Content, nil
    }
    return "", nil
}
```

**Key changes:**
- Use `llms.GenerateContent` instead of `llms.GenerateFromSinglePrompt`
- Streaming callback is passed directly to langchaingo
- Non-streaming mode extracts complete text from `response.Choices`

### StreamWriter for Thread-Safe Output

**File:** `internal/enricher/processor/stream_writer.go`

```go
// StreamWriter ensures thread-safe streaming with batch prefix
type StreamWriter struct {
    mu     sync.Mutex
    writer io.Writer
}

func NewStreamWriter(w io.Writer) *StreamWriter {
    return &StreamWriter{writer: w}
}

func (sw *StreamWriter) WriteWithPrefix(prefix string, chunk []byte) error {
    sw.mu.Lock()
    defer sw.mu.Unlock()

    _, err := fmt.Fprintf(sw.writer, "[%s] ", prefix)
    if err != nil {
        return err
    }
    _, err = sw.writer.Write(chunk)
    return err
}
```

### BatchProcessor with Streaming Support

**File:** `internal/enricher/processor/batch.go`

```go
// BatchProcessor processes batches of elements
type BatchProcessor struct {
    provider     provider.Provider
    templateMgr  *prompt.TemplateManager
    streamWriter *StreamWriter
}

// BatchProcessorOption for configuring BatchProcessor
type BatchProcessorOption func(*BatchProcessor)

// WithStreamWriter sets streaming writer for BatchProcessor
func WithStreamWriter(sw *StreamWriter) BatchProcessorOption {
    return func(bp *BatchProcessor) {
        bp.streamWriter = sw
    }
}

// ProcessBatch processes a single batch with optional streaming
func (p *BatchProcessor) ProcessBatch(ctx context.Context, batch *Batch) error {
    tmpl, err := p.templateMgr.Get(batch.Type)
    if err != nil {
        return fmt.Errorf("failed to get template: %w", err)
    }

    prefix := string(batch.Type) // "API", "Schema", "Param"

    for _, elem := range batch.Elements {
        systemPrompt, userPrompt, err := tmpl.Render(elem.Context)
        if err != nil {
            slog.Warn("failed to render template", "error", err, "path", elem.Path)
            continue
        }

        fullPrompt := systemPrompt + "\n\n" + userPrompt

        var genOpts []provider.Option
        if p.streamWriter != nil {
            genOpts = append(genOpts, provider.WithStreamingFunc(func(ctx context.Context, chunk []byte) error {
                return p.streamWriter.WriteWithPrefix(prefix, chunk)
            }))
        }

        response, err := p.provider.Generate(ctx, fullPrompt, genOpts...)
        if err != nil {
            return fmt.Errorf("LLM call failed: %w", err)
        }

        // Handle response parsing (existing logic)
        // ...
    }

    return nil
}
```

### Enricher Options

**File:** `internal/enricher/enricher.go`

```go
// EnrichOptions provides runtime options for enrichment
type EnrichOptions struct {
    Language string
    Stream   *bool     // Enable streaming output (nil = default true, explicit true/false overrides)
    Writer   io.Writer // Optional: custom writer for streaming (default: os.Stdout)
}

// Enrich enriches an OpenAPI spec with AI-generated descriptions
func (e *Enricher) Enrich(ctx context.Context, spec *openapi3.T, opts *EnrichOptions) (*openapi3.T, error) {
    // ... existing logic ...

    var streamWriter *processor.StreamWriter
    stream := true // default: streaming enabled
    writer := os.Stdout // default
    if opts != nil {
        // Stream is tri-state (*bool): nil means "use default" (true)
        if opts.Stream != nil {
            stream = *opts.Stream
        }
        if opts.Writer != nil {
            writer = opts.Writer
        }
    }

    if stream {
        streamWriter = processor.NewStreamWriter(writer)
    }

    batchProcessor := processor.NewBatchProcessor(e.provider, tmplMgr,
        processor.WithStreamWriter(streamWriter))
    concurrentProcessor := processor.NewConcurrentProcessor(batchProcessor, e.config.Concurrency)

    // ... rest of processing ...
}
```

## Output Example

```
[api] {"description": "用户管理接口
[api] ，用于查询系统
[api] 中的用户列表。"}
[schema] {"id": "用户唯一
[schema] 标识符", "name":
[schema] "用户名称"}
...
```

**Note:** The streaming output shows raw LLM response chunks with type prefixes (`[api]`, `[schema]`, `[param]`). The prompt templates request JSON output, so the streamed content is JSON being assembled incrementally. This provides real-time feedback during enrichment without waiting for complete responses.

## Data Flow

```
CLI (--no-stream=false)
    ↓ EnrichOptions{Stream: true}
Enricher.Enrich()
    ↓ StreamWriter(os.Stdout)
ConcurrentProcessor.ProcessAll()
    ↓ Concurrent + Mutex
BatchProcessor.ProcessBatch()
    ↓ provider.WithStreamingFunc
Provider.Generate()
    ↓ llms.WithStreamingFunc
langchaingo LLM
    ↓ chunk by chunk
StreamWriter.WriteWithPrefix("[api] ")
    ↓
stdout (real-time output)
```

## Dependencies

- `github.com/tmc/langchaingo v0.1.14` (already in use)

## Risks and Mitigations

| Risk | Mitigation |
|------|------------|
| Output interleaving with high concurrency | StreamWriter mutex + batch prefix |
| Provider implementations diverge | Use langchaingo consistently across all providers |
| Breaking existing API | `opts ...Option` is backward compatible |

## Success Criteria

- [ ] Real-time streaming output visible in terminal
- [ ] Batch prefixes distinguish different enrichment types
- [ ] `--no-stream` flag disables streaming
- [ ] All existing tests pass
- [ ] New streaming tests cover thread safety
